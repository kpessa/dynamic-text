# Story 1.5: Fix Import/Export Functionality

## Status
Draft

## Story
**As a** User,
**I want** To reliably export my work into the required JSON format and import a configuration file,
**so that** I can share my work and ensure compatibility

## Acceptance Criteria
1. Export must generate a JSON file correctly representing all sections.
2. Import must correctly parse a valid JSON file and load the content into the editor.
3. The process must handle the special delimiters for dynamic text (`[f( ... )]`) correctly.

    "populationType": "NEO|CHILD|ADOLESCENT|ADULT",

```

]
```

### Dynamic Text Delimiter Handling
**Delimiter Format** [Source: data-models.md]:
- Pattern: `[f( ... )]`
- Content: JavaScript code that can access `me` object
- Example: `[f( return me.getValue('DoseWeightKG') * 2; )]`
- Must be preserved exactly during import/export

### File Operations
**Download Implementation** [Source: ExportModal.svelte]:
```typescript
function downloadFile(content: string, filename: string, mimeType: string) {
  const blob = new Blob([content], { type: mimeType });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = filename;
  a.click();
  URL.revokeObjectURL(url);
}
```

**Upload Implementation** (Needed):
```typescript
function handleFileUpload(file: File): Promise<string> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = (e) => resolve(e.target.result as string);
    reader.onerror = reject;
    reader.readAsText(file);
  });
}
```
