# Story 1.1: Stabilize Firebase Save/Load Functionality

## Status
Complete

## Story
**As a** User,
**I want** To save my work to Firebase and load it back without losing data or state,
**so that** I can trust the system to persist my work

## Acceptance Criteria
1. When I click "Save", the entire state (sections, tests, etc.) must be saved to Firebase.
2. When I load a configuration from Firebase, the editor must be perfectly restored to its last saved state.
3. The UI must provide clear feedback on the save status (e.g., "Last saved at...").

```typescript
interface Reference {
  id: string;
```

**Section Model** [Source: architecture/data-models.md]:
```typescript
interface Section {
  id: string;
  type: 'static' | 'dynamic';
  name: string;
  content: string;
  order: number;
  testCases?: TestCase[];
  isActive: boolean;
}
```

### Firebase Configuration
**Firestore Document Structure** [Source: architecture/database-schema.md]:
- Collection: `references` for public references
- Collection: `users/{userId}/references` for private references
- Document fields must match Reference model exactly
- Timestamps use Firestore Timestamp type

**Security Rules** [Source: architecture/database-schema.md]:
- Public references: Anyone can read, only owner can write/delete
- Private references: Only owner can read/write
- Authentication required for all write operations

### API Specifications
**FirebaseDataService Methods** [Source: architecture/components.md]:
```typescript
interface FirebaseDataService {
  saveReference(ref: Reference): Promise<void>
  loadReference(id: string): Promise<Reference>
  listReferences(): Promise<Reference[]>
  authenticate(): Promise<User>
  subscribeToChanges(callback): Unsubscribe
}
```

**Required Firestore Operations** [Source: architecture/api-specification.md]:
- `create()` - Save new reference
- `update()` - Modify existing reference  
- `get()` - Retrieve by ID
- `list()` - Query user's references
- `onSnapshot()` - Real-time subscription
